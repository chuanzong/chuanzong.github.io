将实际设备stl文件，完整呈现在网页中

2017.02 -- 2017.09

1、右手坐标系

2、场景(Scene)

（鉴于电脑性能，实际运行环境，可使用setTimeout控制执行频率）

下面我们用一个简单的例子来梳理一下这个过程。

var cube, renderer, scene, camera; 

// renderer首先初始化渲染器:

renderer = new THREE.WebGLRenderer({ alpha: false ,antialias: !1});

renderer.setSize( three_Box.getBoundingClientRect().width, three_Box.getBoundingClientRect().height );

three_Box.appendChild( renderer.domElement );

// scene 初始化场景:

scene = new THREE.Scene();

// camera 初始化相机：

camera = new THREE.PerspectiveCamera( 19, three_Box.getBoundingClientRect().width / three_Box.getBoundingClientRect().height, 1, 10000 );

camera.position.set( -100, 300, -150 );

camera.up.x = 0;

camera.up.y = 0; //相机朝向--相机上方为y轴

camera.up.z = -1;

camera.lookAt({x:0,y:10,z:0});

// light

var spotLight = new THREE.PointLight( 0xffffff );

spotLight.position.set(-50, 300, -100  );

scene.add( spotLight );

场景中加入物体：

var geometry = new THREE.CubeGeometry(50,50,50);

var material = new THREE.MeshLambertMaterial( { color: 0xFFFFFF } );

cube = new THREE.Mesh( geometry, material );

scene.add( cube );

//渲染到场景中：

renderer.render( scene, camera )


沃迪生产线可视化系统

将实际生产线，抽象具化成，可查看动态模型

2017.02 -- 2017.07

项目简介：

（1）可以由生产管理工作人员在ERP输入各工序生产信息数据，包括生产线名称、产品名称、计划数量、实际生产数量、调令单、批次等，并将以上信息通过网络传输到服务器中，以供管理人员查询、分析和追溯，并在车间现场能过电子广告牌进行展示；

（2）可以实现某种产品在每个工序中的生产时间、检查数、良品数、不良品数量、合格率等数据汇总和查询；

（3）可以进行某种产品多个批次之间生产时间、检查数、（良品数、不良品数量、合格率）等数据的对比分析，包括数据分析和图形展示；

（4）可以对某产品某批次或多批次间产出的异常项目数量进行排列分析；

（5）可以对某产品某批次或多批次间产出的异常项目进行报警追踪；

（6）可以实现日报和月报分析和追溯；

实现路线：

（1）CSS动画的两大组成部分：transition（10+）和animation（10+）。

（2）本项目是以transition+class切换为主，animation补充的动态交互项目。

（3）CSS是没有时间轴的。也就是说，所有的状态变化，都是即时完成。

transition的优点在于简单易用，但是它有几个很大的局限。

（1）transition需要事件触发，所以没法在网页加载时自动发生。

（2）transition是一次性的，不能重复发生，除非一再触发。

（3）transition只能定义开始状态和结束状态，不能定义中间状态，也就是说只有两个状态。

（4）一条transition规则，只能定义一个属性的变化，不能涉及多个属性。

css animation就是为了解决这些问题的。

项目总结：

经过这个项目的独立开发，对JavaScript的使用有了更深刻的理解，很大程度的提高了自己实践能力和编程思想以及逻辑思维能力,同时也加深了运用数学算法来实现更加复杂的JS动画的运用。
